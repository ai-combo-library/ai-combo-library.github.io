开发与 Agent · 实测通过 · 有使用限制 · 适合特定人群有实测证据含限制说明
actionlintcodex-plugin-cc
适合谁维护 GitHub Actions、希望 Agent 修复后仍有确定性校验的开发者。
不适合谁没有 Codex 登录态，或 CI 系统并非 GitHub Actions 的团队。
最大的限制需要本机 Codex app-server 登录态；本轮只验证一类工作流字段错误，没有覆盖密钥、权限和真实云端执行失败。

我们怎么测

先向 actionlint 提交一份含有 `runz` 错误字段的真实 workflow，保存其非零退出报告；再让 codex-plugin-cc 读取同一报告、修改同一文件，并重新运行 actionlint。

实际结果

actionlint 首次准确发现错误，Codex 插件把 `runz` 修正为合法字段，第二次 actionlint 返回通过；不是生成替代文件，而是在原工作区完成修复。

实际工作流怎么走

  1. 1. actionlint：整理上一环节的内容，转换成后续工具可以继续读取的结果。
  2. 2. codex-plugin-cc：接收上一步结果并自动继续处理。

继续补成可长期使用方案要做什么

  1. 固定输入样本和预期输出。
  2. 锁定依赖版本并在干净环境重新执行。
  3. 保留失败原因、产物和复测日期。
