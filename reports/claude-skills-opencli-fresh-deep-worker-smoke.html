一句话结论

用 Claude Skill 生成结构稳定的任务交接文档，再由 OpenCLI 在命令行读取、检查并接入下一段自动化。

我们怎么测

运行 handoff Skill 生成包含目标、当前状态、待决事项、所需 Skills 和产物的五段式 Markdown，再由真实 OpenCLI 本地扩展读取该文件并解析关键字段。

实际结果

OpenCLI 成功读到交接目标、5 个结构段和产物区，最终输出被判定为可继续执行的有效交接，而不是只有一份无法调用的文档。

实际工作流怎么走

  1. 1. claude-skills（处理/理解）：接收 task；source material，产出 skill instructions；cli command。
  2. 2. OpenCLI（自动执行）：接收 cli command；skill instructions，产出 execution result；file。

继续补成可长期使用方案要做什么

  1. 固定输入样本和预期输出。
  2. 锁定依赖版本并在干净环境重新执行。
  3. 保留失败原因、产物和复测日期。
