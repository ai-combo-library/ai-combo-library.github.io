# FunClip + VideoCaptioner 轻量视频字幕实测

## 结论

组合链路跑通。

本轮在本机完成了：

1. 生成 9.6 秒测试视频，包含英文语音音轨。
2. FunClip 对视频做 ASR，输出 `total.srt`、原始识别文本、句子时间戳。
3. VideoCaptioner 读取 FunClip 产出的 `total.srt`，把字幕软合成进 mp4。
4. `ffprobe` 确认最终视频包含 `mov_text` 字幕轨。

因此这条组合可以从 L0 理论匹配升级到 L3 最小闭环实测。

## 实测结果

### 输入视频

- 路径：`input/sample-video.mp4`
- 时长：9.6 秒
- 视频：H.264, 1280x720, 25fps
- 音频：AAC, mono

### FunClip 输出

- `output/funclip/total.srt`
- `output/funclip/recog_res_raw`
- `output/funclip/sentences`
- `output/funclip/timestamp`

FunClip 识别结果可用，但质量一般：

```text
this be the short test video for AI subtite workflow.
```

其中 `this is` 被识别成 `this be`，`subtitle` 被识别成 `subtite`。

### VideoCaptioner 输出

- 直接转录测试：`output/videocaptioner/sample-video.srt`
- 用自己转录结果合成：`output/videocaptioner/sample-video-captioned-soft.mp4`
- 用 FunClip SRT 合成：`output/videocaptioner/funclip-srt-captioned-soft.mp4`

最终 `ffprobe` 显示输出视频包含第三条字幕流：

```text
codec_name: mov_text
codec_type: subtitle
duration: 9.660000
```

## 关键发现

### 1. 链路真实成立

`FunClip → SRT → VideoCaptioner → 带字幕视频` 这条链路成立。

格式衔接没有问题，SRT 是标准连接点。

### 2. 安装成本比原先评分更高

原先记录里写“pip 安装即用”不准确。

- FunClip 不是 PyPI 包，`pip install funclip` 不存在。
- FunClip 需要 GitHub clone 后安装 `requirements.txt`。
- VideoCaptioner 只支持 Python `>=3.10,<3.13`，本机默认 Python 3.14 不能安装。
- 本轮额外通过 Homebrew 安装了 Python 3.12。
- FunClip 会下载约 823MB 的英文 ASR 模型。

### 3. 两个工具不建议放在同一个 Python 环境

FunClip requirements 会把 `pillow` 降到 `10.4.0`，而 VideoCaptioner 需要 `pillow>=12.0.0`。

本轮虽然 VideoCaptioner 的软字幕合成仍能跑通，但正式测试协议应使用两个独立 venv：

- `venv-funclip`
- `venv-videocaptioner`

### 4. VideoCaptioner CLI 比预期成熟

VideoCaptioner 提供完整 CLI：

- `transcribe`
- `subtitle`
- `synthesize`
- `process`
- `dub`
- `download`
- `doctor`

这比只依赖 GUI 更适合作为组合技的一环。
