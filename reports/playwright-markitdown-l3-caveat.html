PlaywrightMarkItDown
适合谁适合需要整理网页资料、但普通 HTTP 抓取拿不到完整正文的人，尤其是轻量研究、资料归档和内容整理场景。
不适合谁不适合登录态复杂、强反爬、无限滚动很多，或要求大规模并发采集的网站。

我们怎么测

复核此前长期使用运行中由 Playwright 渲染的动态 HTML 和 MarkItDown 生成的 Markdown，并检查只有 JavaScript 执行后才出现的动态文本是否被保留。

实际结果

渲染 HTML 与 Markdown 产物均存在，动态信号进入最终 Markdown，4 项检查通过；本轮属于既有运行产物复核，没有重新覆盖登录态和反爬页面。

实际工作流怎么走

  1. 1. Playwright：获取原始资料，并把内容交给下一步。
  2. 2. MarkItDown：整理上一环节的内容，转换成后续工具可以继续读取的结果。

继续补成可长期使用方案要做什么

  1. 固定测试样本和最终产物
