@playwright/mcp + chrome-devtools-mcp

让 Playwright MCP 重放真实页面交互，再由 Chrome DevTools MCP 接手同一页面的网络与性能证据，适合 Agent 自动验收网页。

开发与 Agent · 实测通过 · 有使用限制 · 强推荐有实测证据含限制说明
@playwright/mcpchrome-devtools-mcp
适合谁让 Agent 执行网页回归、排查性能和网络问题的开发者与测试团队。
不适合谁只需要简单截图，或目标站点禁止自动化访问的用户。
最大的限制本轮使用本地诊断页，未覆盖登录站点、跨域限制、验证码和长时间性能采样。

我们怎么测

Playwright MCP 先打开本地诊断页并完成按钮交互，再把页面 URL 和执行轨迹交给 Chrome DevTools MCP；后者连接同一页面，读取网络与性能信息。

实际结果

两个 MCP 服务都通过官方协议真实启动，交互轨迹成功交接，DevTools 侧拿到页面诊断结果，功能断言与最终诊断断言均通过。

实际工作流怎么走

  1. 1. @playwright/mcp：获取原始资料，并把内容交给下一步。
  2. 2. chrome-devtools-mcp：整理上一环节的内容，转换成后续工具可以继续读取的结果。

继续补成可长期使用方案要做什么

  1. 固定输入样本和预期输出。
  2. 锁定依赖版本并在干净环境重新执行。
  3. 保留失败原因、产物和复测日期。
