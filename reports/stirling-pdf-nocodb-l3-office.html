不适合谁不适合要求完整 UI 路径、零配置上线或马上投入生产的用户。
这次确认了什么两个工具在同一次实测中都已运行，交接断言通过 6/6。
最大的限制不是所有 PDF 都能抽出 CSV 表格；Markdown 返回 ZIP，正式自动化需要解包；Stirling-PDF 默认认证，NocoDB API 需使用 xc-auth。

我们怎么测

重新启动 Stirling-PDF 与 NocoDB 容器，用 Stirling API 处理 PDF，并检查此前通过 NocoDB API 写入的页数、摘要和文件信息是否仍能从持久化台账读回。

实际结果

两个服务均正常启动，Stirling API 可重新执行，NocoDB 中的台账行仍可读取，6 项检查通过；因旧 token 未保留，本轮没有再通过 HTTP API 新写一行。

实际工作流怎么走

  1. 1. Stirling-PDF（处理/理解）：接收 PDF 文件，产出 PDF 基础信息, 页数, Markdown ZIP, 图片, CSV（依文档结构）。
  2. 2. nocodb（输出/交付）：接收 JSON, HTTP API, structured records，产出 在线表格, 数据库记录, API。
