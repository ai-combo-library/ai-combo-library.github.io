Stirling-PDF + second-brain

把 PDF 从静态文件变成可被本地 AI 助手读取、引用和继续处理的知识资产。

个人生活与财务 · 实测通过 · 有使用限制 · 适合特定人群有实测证据含限制说明
Stirling-PDFsecond-brain
适合谁适合希望把 PDF 资料先处理成可读文本，再沉淀进本地 AI 助手的个人或小团队。
不适合谁不适合完全不想部署 Docker、需要企业级权限审计，或要求马上完成语义索引检索的人。
最大的限制本轮验证的是 Stirling-PDF 本地服务接口将 PDF 转 Markdown 压缩包，并通过 second-brain 核心程序的附件解析功能接入 AI 可读取的文本；未安装 second-brain 完整功能包、未测试语义搜索和长期资料检索，也未测试复杂表格 PDF。

我们怎么测

启动 Stirling-PDF Docker 服务，把两页 PDF 通过 API 转成 Markdown 压缩包；解包后分别把 Markdown 和原 PDF 交给 second-brain 附件解析层，并检查生成的 解析后的文本 与 LLM 上下文后缀。

实际结果

Stirling-PDF 生成了非空 Markdown，second-brain 同时接收 Markdown/PDF 并产出 LLM 可读内容，6 项检查通过；本轮没有继续验证 语义向量服务 和长期检索。

实际工作流怎么走

  1. 1. Stirling-PDF：整理上一环节的内容，转换成后续工具可以继续读取的结果。
  2. 2. second-brain：接收上一步结果并自动继续处理。

继续补成可长期使用方案要做什么

  1. 固定测试样本和最终产物
  2. 用同样样本重新跑一遍完整流程
  3. 保留失败原因
  4. 保存完整实测记录
  5. 定期重新确认是否仍然可用
