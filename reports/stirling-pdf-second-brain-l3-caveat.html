一句话结论

把 PDF 从静态文件变成可被本地 AI 运行环境 读取、引用和继续处理的知识资产。

我们怎么测

启动 Stirling-PDF Docker 服务，把两页 PDF 通过 API 转成 Markdown ZIP；解包后分别把 Markdown 和原 PDF 交给 second-brain 附件解析层，并检查生成的 parsed_text 与 LLM 上下文后缀。

实际结果

Stirling-PDF 生成了非空 Markdown，second-brain 同时接收 Markdown/PDF 并产出 LLM 可读内容，6 项检查通过；本轮没有继续验证 embedding 和长期检索。

实际工作流怎么走

  1. 1. Stirling-PDF（处理/理解）：接收 pdf，产出 pdf。
  2. 2. second-brain（自动执行）：接收 markdown, pdf, slides, spreadsheets, archives, images, audio, video, url，产出 json, markdown, api。

继续补成可长期使用方案要做什么

  1. 固定输入输出
  2. 运行 deep rerun
  3. 保留失败原因
  4. 进入证据中心
  5. 定期复测
