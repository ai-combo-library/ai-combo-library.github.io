一句话结论

让 Supabase Agent Skills 按项目规范生成数据库迁移，再由 Supabase CLI 在本地数据库真正执行并查询验证。

我们怎么测

先依据 Supabase Agent Skills 生成带表结构和种子数据的 migration，然后启动本地 Supabase 栈，把 SQL 放进数据库容器用 psql 执行，并查询新表。

实际结果

迁移在真实本地 PostgreSQL 中执行成功，查询返回预期记录；验证的是可落地数据库变更，不是只检查 SQL 文本格式。

实际工作流怎么走

  1. 1. Supabase Agent Skills（处理/理解）：接收 database task；schema requirement，产出 sql；cli command；skill instructions。
  2. 2. Supabase CLI（自动执行）：接收 sql；cli command，产出 database；schema diff；execution result。

继续补成可长期使用方案要做什么

  1. 固定输入样本和预期输出。
  2. 锁定依赖版本并在干净环境重新执行。
  3. 保留失败原因、产物和复测日期。
