我们怎么测

通过 UI5 MCP Server 创建 OpenUI5 TypeScript 应用，运行 manifest 校验和 UI5 linter；启动应用后由 Playwright 打开页面、点击按钮并检查对话框。

实际结果

应用成功生成，manifest 与 linter 均通过；Playwright 能加载页面、触发按钮、看到对话框并保存截图，生成与浏览器验收形成完整闭环。

实际工作流怎么走

  1. 1. UI5 MCP Server（处理/理解）：接收 ui5 project spec；source code，产出 web app；url；validation report。
  2. 2. Playwright（输出/交付）：接收 web app；url，产出 test result；screenshot；console log。

继续补成可长期使用方案要做什么

  1. 固定输入样本和预期输出。
  2. 锁定依赖版本并在干净环境重新执行。
  3. 保留失败原因、产物和复测日期。
